## 2. Análise de Banco de Dados

### Versões Utilizadas  
- PostgreSQL 13.7 (produção prioritária).  
- Oracle Database 19c (integrações legadas).  
- MySQL 8.0 (aplicações menores/testes).  
Planejamento para PostgreSQL 15 até Q3/2024 para garantir suporte e modernidade.

### Replicação e Backup  
- Replicação streaming assíncrona com Patroni no PostgreSQL.  
- Oracle 19c com Data Guard standby físico.  
- MySQL com Group Replication em cluster.  
- Backups full diários e incrementais a cada 4 horas, via pgBackRest (PostgreSQL), RMAN (Oracle), MySQL Enterprise Backup, com armazenamento off-site/cloud e testes mensais de restauração.

### Segurança  
- Criptografia em repouso: TDE Oracle, tablespace criptografado MySQL, criptografia no PostgreSQL.  
- Comunicação segura via TLS 1.2+.  
- Autenticação via LDAP+SSL, controle de acesso baseado em privilégios mínimos e MFA para acessos privilegiados.  
- Auditoria ativada com pgAudit (PostgreSQL), Unified Auditing (Oracle) e integração com SIEM para detectar acessos anômalos.

### Monitoramento e Tuning de Desempenho  
- Métricas monitoradas por Zabbix e Prometheus, com alertas configurados.  
- Dashboards e análises preditivas de comportamento.  
- Tuning por análise dos planos de execução, aplicação de índices compostos, particionamento e revisão periódica da parametrização dos bancos (ex: max_connections, work_mem, shared_buffers no PostgreSQL).

### Sugestões de Melhoria  
- Atualizar bancos legados para versões suportadas.  
- Implementar criptografia avançada para tabelas sem proteção atual.  
- Automatizar runbooks de recuperação de desastres.  
- Reforçar testes de backup com simulações de restauração cruzada entre bancos.
